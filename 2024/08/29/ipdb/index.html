（ai生成）ipdb调试手册

快速上手

首先pip install ipdb，在需要的文件中import ipdb，需要停顿的地方塞上ipdb.set_trace()。正常地python main.py后运行到那一行就能正常进入调试。

  1. 查看当前代码位置：
    • l 或 list：列出当前位置的代码。默认情况下，它会显示当前行前后几行代码，但你可以通过提供行号来查看特定区域的代码，例如 list 10。
  2. 单步执行：
    • n 或 next：执行下一行代码，但不进入任何函数调用。
    • s 或 step：进入当前行的函数调用（如果当前行是一个函数调用）。
  3. 继续执行：
    • c 或 continue：继续执行程序，直到遇到下一个断点或程序结束。
    • j 或 jump：跳转到指定的行号。
  4. 检查变量：
    • p 或 print：打印变量的值。例如，p x 会打印变量 x 的值，p x.shape 会打印 x 的形状。
    • pp：美化打印（pretty print），用于更易读的格式显示复杂数据结构。
  5. 设置和清除断点：
    • b 或 break：设置断点。例如，break 10 在第 10 行设置断点。
    • cl 或 clear：清除断点。例如，clear 10 清除第 10 行的断点。
  6. 查看当前上下文：
    • f 或 frame：显示当前栈帧的信息。
    • u 或 up：移动到更高级别的栈帧（向上查看调用栈）。
    • d 或 down：移动到更低级别的栈帧（向下查看调用栈）。
  7. 查看和修改变量：
    • vars：列出当前上下文中的所有变量。
    • 直接赋值：你可以在控制台中直接对变量进行赋值，例如 x = 10。
  8. 退出调试器：
    • q 或 quit：退出调试器并结束程序。
    • exit：退出调试器并结束程序。
  9. 运行 Python 命令：
    • 直接在 ipdb 提示符下输入 Python 代码：你可以执行任意 Python 表达式来检查程序状态。
  10. 查看当前文件和行号：
    • w 或 where：显示当前执行的栈跟踪，包括文件名和行号。
